## 전통적인 웹 애플리케이션

- 서버에 페이지 단위로 요청을 보내며 웹서버는 완성된 페이지(HTML)를 응답
- 필요한 데이터만 갱신하고 싶어도 전체 페이지를 새로 받음
  - 리플래시(화면 깜빡이며 다시 렌더링), 렌더링 비용 증가
  - 사용자 경험(UX)이 좋지 않음




<br />
<br />




## MPA(Multi Page Application)

- 기본 구조는 전통적인 방식과 동일하게 페이지 단위 요청
- 차이점으로 API로 요청하여 페이지 내부의 일부만 부분 갱신 가능
- 부분 갱신 덕분에 전체 리플래시가 줄어 UX 개선
- 여전히 페이지마다 별도 HTML이 존재하며, 레이아웃 전체가 바뀌는 시점에는 리플래시가 발생




<br />
<br />




## SPA(Single Page Application)

- 하나의 HTML(index.html)을 기반으로 필요한 데이터만 API로 요청하여 화면 전환
- 화면 전환시 리플래시가 없어 UX 향상
- 서버 부담 감소, 클라이언트(브라우저) 중심으로 개발 가능
- 한 페이지에서 구현하다 보니 상태(테이터, 변수) 관리가 어려움
- DOM을 자주 갱신할 수록 성능 저하
- HTML을 자바스크립트에서 생성하므로 개발 생산성 저하

<br />

### React 특징

- React는 상태(데이터)가 변경되면 반응(랜더링)하는 방식으로 동작
- SPA의 단점을 해결할 수 있는 라이브러리
- JSX로 HTML 작성 생산성 향상
- 가상 DOM으로 성능 저하 최소
- 컴포넌트별로 상태 관리가 용이하며, 재사용성과 유지보수성 향상
- 전역 상태 관리를 위한 라이브러리가 많음 (zustand, recoil 등)

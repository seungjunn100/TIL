## Redux

- `Redux`는 `JavaScript` 애플리케이션의 상태(`state`)를 관리하기 위한 라이브러리다.

- 여러 컴포넌트가 공유하는 상태를 하나의 중앙 저장소(`Store`)에 두고 정해진 규칙으로 관리한다.

- `React` 자체는 `UI` 라이브러리고, `Redux`는 상태 변경을 규칙과 절차 안에서만 하도록 강제하는 설계도 같은 도구이다.

<br />

### Redux 특징

- 애플리케이션의 모든 상태를 하나의 저장소에서 관리한다.

- 상태 변경이 명확한 규칙을 따르므로, 버그를 찾고 디버깅하기 쉽다.

  - 상태 변경은 항상 순수 함수(`Reducer`)를 통해 이루어지며, 동일한 입력에는 동일한 출력이 나오므로 예측이 가능하다.

- 데이터는 컴포넌트(이벤트 발생) → 액션(작업 정의) → 리듀서(상태 관리) → 스토어(상태 저장) 순으로 한 방향으로 흐른다.

## Flux 아키텍처

단방향 데이터 흐름을 강제하여 애플리케이션 구조를 단순화한다.

<br />

### Flux 구성 요소

![Flux 구성 요소](../src/images/flux.png)
*출처 - "원쌤의 리액트 퀵스타트 with 타입스트립트"*

#### Action

- 무슨 일이 일어났는지 설명하는 객체

- 예를 들어, 사용자가 버튼을 눌렀을 때 발생하는 이벤트

```tsx
{ type: 'countDown', payload: { step: 2 } }
```

#### Dispatcher

- `Action`을 받아서 `Store`로 전달하는 중앙 허브 역할

#### Store

- 애플리케이션의 상태와 상태를 변경하는 로직을 담고 있다.

  - `Action`을 받아 상태 변경

- `Flux`에서는 여러 개의 스토어를 가질 수 있다.

- 각 스토어는 특정 도메인의 데이터를 관리합니다.

#### View

- `Store`의 상태를 읽어 `UI`를 렌더링

- 사용자 이벤트 발생 시 `Action` 생성

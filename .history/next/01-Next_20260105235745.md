## 리액트와 Next.js의 차이

### 리액트의 한계

- `UI` 라이브러리

- 초기 번들 용량이 커 첫 화면 로딩이 오래 걸린다.

- 코드 분할, lazy loading 등 성능 최적화를 직접 구현해야 한다.

- `CSR(Client-Side Rendering)` 기반이라 `SEO(Search Engine Optimization, 검색 엔진 최적화)`에 불리하다.
  
  - 메타데이터, 링크 등 페이지에 대한 정보가 하나도 없는 빈 `HTML`을 초기에 응답 받으므로 검색엔진이 읽을 컨텐츠가 없다.

- 백엔드 API를 별도로 구축해야 함

<br />

### Next.js의 장점

- `SSR(Server-Side Rendering)`로 초기 로딩 속도가 개선된다.

  - 완성된 `HTML`을 응답하므로 즉시 화면에 표시가 가능하다.

    - 검색 엔진이 컨텐츠를 바로 읽을 수 있어 `SEO`를 최적화 할 수 있다.

- 파일 시스템 기반 라우팅 자동 생성

- 자동 코드 분할, 이미지/폰트 최적화 내장

- `API Routes`로 풀스택 개발 가능

- 다양한 렌더링 전략 지원
  
  - `SSR`
  - `SSG(Static Site Generation, 사이트 정적 생성)`
  - `ISR(Incremental Static Regeneration, 증분 정적 재생성)`

## Next.js란

- 리액트 기반의 풀스택 웹 애플리케이션을 구축하기 위한 프레임워크

- 프레임워크

  - 소프트웨어 개발에 필요한 공통 구조와 기능 제공

  - 개발 방식이나 프로젝트 구조를 강제하여 자유도는 낮지만, 제공되는 공통 기능을 활용하면 개발 생산성 향상
  
<br />

### 개발 환경 구성

![개발 환경 구성](../src/images/nextjs-env-1.png)

- `npm init -y`을 실행하여 새로운 `Node.js` 프로젝트를 만들고, `package.json` 파일을 생성한다.

- 필요한 `Node` 패키지를 설치한다.

- `App` 라우터를 사용하기 위해 `app` 디렉토리를 생성한다.

![개발 환경 구성](../src/images/nextjs-env-2.png)

- 리액트의 `children` 규칙을 따르면서, `Readonly`를 사용하여 `props`를 수정하는 실수를 타입 레벨에서 방지한다.

- 루트 레이아웃은 공통의 컴포넌트이고 `children`에는 파일들의 루트 페이지를 받아 렌더링하게 된다.

![Next.js 라우팅](../src/images/nextjs-router.png)

- 파일 시스템 기반의 라우터를 제공하여 파일의 경로대로 라우팅된다.

![개발 환경 구성](../src/images/nextjs-env-3.png)

```node
npx create-next-app@latest
```
- 위의 코드를 실행하여 `Next.js` 개발 환경을 자동으로 세팅할 수 있다.

- 기본 값으로 세팅할 수도 있고, 설정 옵션을 지정해서 세팅할 수도 있다.

![개발 환경 구성](../src/images/nextjs-env-4.png)
